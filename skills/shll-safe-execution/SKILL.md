# SHLL Safe Execution

Contract-enforced DeFi execution layer for AI agents on BNB Chain (BSC). Every trade passes through on-chain PolicyGuard validation before execution.

## What This Skill Does

This skill enables AI agents to execute DeFi operations (swap, wrap, unwrap, transfer) with **mandatory on-chain policy enforcement**. PolicyGuard validates every action at the smart contract level — the AI cannot bypass or disable safety checks.

**Key difference from other trading skills:** Safety validation is not optional middleware. It is enforced by the smart contract itself via `revert`. Even if the AI agent is compromised, policy violations are blocked on-chain.

## Security Architecture

### Dual-Wallet Isolation

| Wallet | Controller | Capabilities |
|---|---|---|
| **Owner** | User (human) | Asset control, policy configuration, vault admin |
| **Operator** | AI agent | Execute trades within policy boundaries only |
| **Vault** | Smart contract (ERC-6551) | Holds all agent funds, isolated from both wallets |

The operator wallet (AI) **cannot** transfer funds out of the vault to arbitrary addresses. All outbound value flows through PolicyGuard.

### 4-Policy Stack

| Policy | What It Enforces |
|---|---|
| **SpendingLimitV2** | Per-transaction and daily spending caps |
| **CooldownPolicy** | Minimum time interval between trades |
| **DeFiGuardV2** | Whitelist of approved DeFi protocols and function selectors |
| **ReceiverGuardV2** | Whitelist of approved recipient addresses |

All policies are configured by the **owner wallet** and enforced by the smart contract. The AI agent has no capability to modify them.

## Supported Operations

### Swap (PancakeSwap V2/V3)

```bash
shll-run swap -f BNB -t USDT -a 0.01 -k <tokenId>
```

Swaps tokens via PancakeSwap with automatic routing. PolicyGuard validates:
- Amount against SpendingLimitV2
- DEX router against DeFiGuardV2
- Time since last trade against CooldownPolicy

### Wrap / Unwrap (BNB ↔ WBNB)

```bash
shll-run wrap -a 0.1 -k <tokenId>
shll-run unwrap -a 0.1 -k <tokenId>
```

### Transfer

```bash
shll-run transfer --token USDT -a 10 --to <address> -k <tokenId>
```

Transfers tokens from the vault. ReceiverGuardV2 validates the recipient address is on the approved list.

### Cross-Skill Execution (MCP)

For transaction payloads generated by external aggregators (OKX, 1inch, etc.):

```
MCP Tool: execute_calldata
  - tokenId: <agentTokenId>
  - target: <contract address>
  - calldata: <hex payload from other skill>
```

PolicyGuard decodes and validates the calldata before execution. If any policy is violated, the transaction reverts with a specific reason.

## Success vs Rejection Behavior

### Within Policy → Approved

```json
{
  "status": "success",
  "txHash": "0x3a7f...c218",
  "amountIn": "0.01 BNB",
  "amountOut": "5.82 USDT"
}
```

### Exceeds Policy → Rejected On-Chain

```json
{
  "status": "rejected",
  "reason": "SpendingLimitV2: amount 0.05 BNB exceeds per-transaction limit of 0.02 BNB"
}
```

Rejections are **on-chain reverts** — no funds are moved, no gas is consumed beyond the failed call.
